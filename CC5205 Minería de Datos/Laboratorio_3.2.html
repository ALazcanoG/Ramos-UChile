Parte 1: Teoría¶

1. De acuerdo a lo visto en clases, explique cómo bisecting k-mean soluciona el problema de la inicialización de centroides de k-means.

Respuesta: En k-mean, los centroides son escogidos (usualmente) de forma aleatoria por lo que, a pesar de ejecutar el algoritmo varias veces, podemos no obtener el óptimo deseado. Así es como, debido a la naturaleza de bisecting k-means, este tiene menos problemas de inicializar centroides. Esto pues el conjunto de los datos es dividido en solo 2 clusters en cada iteración escogiendo el que tenga menor SSE.

2. Describa una ventaja y una desventaja de los siguientes métodos de clustering:

  • K-Means
  • Clustering Jerárquico Aglomerativo

Respuesta: Ventaja K-Means: El algoritmo converge y lo hace usualmente en pocas iteraciones (lo que permite ejecutar varias veces el algoritmo para usar el de menor SSE) y funciona mejor con datos de forma esférica que Clustering Jerárquico Aglomerativo.
Ventaja Clustering Jerárquico Aglomerativo: Más robusto a outliers y no necesita un número de clusters de antemano.
Desventaja K-Means: No es robusto para outliers y se necesita el valor de k de antemano.
Desventaja Clustering Jerárquico Aglomerativo: Necesitan una noción de distancia entre clusters, donde una elección distinta puede variar bastante el resultado.

3. Evaluar clusters no es una tarea fácil. ¿Qué formas de evaluar clusters conoce? Nombre 3 formas de validación y describa cómo podría determinar si los clusters son buenos o malos usando cada una de las formas nombradas.

Respuesta: Formas de validar clusters pueden ser usando correlación, SSE y coeficiente de Silhouette.
En el primero caso, podemos comparar matrices de incidencia y proximidad. En caso en que sean parecidas (calcular correlación) significa que el resultado de clusterización es bueno. En el caso de SSE, podemos generar puntos aleatorios y ejecutar k-means calculando SSE en cada caso y graficarlos en un histograma y comparar con el valor original (la idea es que esté lejos de los valores generador por puntos aleatorios). Por último, para el coef. de Silhouette, es necesario calcularlo y para que sea un buen resultado necesitamos que sea un valor cercano a 1.

4. Según lo visto en clases, explique el enfoque visual para comparar clusters mediante matrices de proximidad. ¿Qué características debe tener la matriz para determinar que una clusterización es buena? ¿En qué casos la matriz de proximidad no es útil para evaluar clustering?

Respuesta: El resultado será una matriz por bloques que, para que sea un buen resultado, la idea es que los bloques en diagonal sean azules-fríos (o el color que represente un número bajo) mientras que los otros bloques sean de colores más cálidos. Este tipo de enfoque es bueno para clusters en los cuales sus puntos están cerca de un centro (por eso no es tan bueno con DBSCAN por ejemplo). Esto para el caso de matriz de proximidad (para matriz de similitud es al revés).

K-Means¶

1. Cuando usamos K-Means debemos definir previamente el número de clusters que queremos generar. Teniendo en cuenta estos datos, implemente el método del codo.

In [2]:
from sklearn.cluster import KMeans
from sklearn import datasets
import matplotlib.pyplot as plt
import numpy as np

sse = []

clusters = list(range(1, 16))
for k in clusters:
    kmeans = KMeans(n_clusters=k).fit(X)
    sse.append(kmeans.inertia_)

plt.plot(clusters, sse, marker="o")
plt.title("Metodo del codo de 1 a 15 clusters")
plt.grid(True)
plt.show()

Teoría¶

1. Describa los componentes principales de un clasificador.¶

Respuesta: Se necesita:

-un conjunto de entrenamientos, donde se conoce la clase real de los datos.

-El algoritmo de clasificacin a utilizar

-Un conjunto de validacin, donde se sabe la clase real de los datos, pero no se ocupar para entrenar el modelo, sino probarlo

2. Un consultor en minería de datos tiene la siguiente estrategia al momento de enfrentar un problema de clasificación: probar todos los modelos posibles hasta encontrar uno que tenga cero error de entrenamiento. Comente la estrategia del consultor y proponga una estrategia alternativa para evaluar el desempeño de los modelos.¶

Respuesta: La estrategia del consultor no es buena, es extremadamente difcil obtener un error de de cero (sino imposible), por lo que no se espera que un clasificador tenga error cero, sino que sea lo menor posible comparado con otros.

Una estrategia alternativa es calcular el error de entrenamiento de cada modelo al hacer cross-validation y elegir aquel que tenga menor error promedio (por ejemplo, el score $F_1$ o ROC AUC)

3. El mismo consultor siempre usa la misma métrica para evaluar un modelo: accuracy. Comente la estrategia del consultor y proponga una estrategia alternativa. Considere las distintas opciones sobre el balance de los datos.¶

Respuesta: El accuracy si nos da informacin util sobre que tan bueno es un modelo, funciona bien cuando hay una distribucin parecida entre las posibles clases (por ejemplo, 50/50 de proporcin entre 2 clases "Si" y "No"), as que la estrategia del consultor si puede servir en este caso.

Sin embargo, en el caso de que se desee predecir una clase que est muy poco presente en comparacin a otra clase, el Accuracy puede resultar incertero.

Por ejemplo si 10% de los datos son del tipo "Con Covid" y 90% "Sin Covid", un clasificador que siempre prediga "No" tendr un Accuracy de 90%, lo cual suena bueno pero no estaramos prediciendo ningn "Covid" cuando esta etiqueta podra ser importante

Para lidiar con esto, se puedo hacer Macro-avering del coeficiente $F_1$ de cada clase posible, esto le da ms representabilidad a las clases que estn menos presentes

CLASIFICADOR 3¶

Para este clasificador utilizaremos Cross-validation. Con cross-validation, el conjunto de entrenamiento se divide en $k$ conjuntos disjuntos; se entrena sobre los datos correspondientes a $k-1$ de stos, y se evala sobre el conjunto restante. Esto se repite $k$ veces, evaluando siempre sobre un conjunto distinto. Teniendo en cuenta el parmetro $k$, a ste mtodo se le llama $k$-fold cross-validation.

El cdigo que est a continuacin realizar 10-fold cross-validation usando Decision Tree sobre los datos. La forma de estimar el rendimiento del clasificador es, entre otras cosas, calculando el promedio de todos los k-folds.

Adems se muestra la predicin sobre los datos para testear.

from sklearn.model_selection import cross_validate
import numpy as np

scoring = ['precision_macro', 'recall_macro', 'accuracy', 'f1_macro']
cv_results = cross_validate(clf, X, y, cv = 10, scoring = scoring, return_train_score= True)

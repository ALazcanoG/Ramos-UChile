Teoría¶

1. Describa los componentes principales de un clasificador.¶

Respuesta: Se necesita:

-un conjunto de entrenamientos, donde se conoce la clase real de los datos.

-El algoritmo de clasificación a utilizar

-Un conjunto de validación, donde se sabe la clase real de los datos, pero no se ocupará para entrenar el modelo, sino probarlo

2. Un consultor en minería de datos tiene la siguiente estrategia al momento de enfrentar un problema de clasificación: probar todos los modelos posibles hasta encontrar uno que tenga cero error de entrenamiento. Comente la estrategia del consultor y proponga una estrategia alternativa para evaluar el desempeño de los modelos.¶

Respuesta: La estrategia del consultor no es buena, es extremadamente difícil obtener un error de de cero (sino imposible), por lo que no se espera que un clasificador tenga error cero, sino que sea lo menor posible comparado con otros.

Una estrategia alternativa es calcular el error de entrenamiento de cada modelo al hacer cross-validation y elegir aquel que tenga menor error promedio (por ejemplo, el score $F_1$ o ROC AUC)

3. El mismo consultor siempre usa la misma métrica para evaluar un modelo: accuracy. Comente la estrategia del consultor y proponga una estrategia alternativa. Considere las distintas opciones sobre el balance de los datos.¶

Respuesta: El accuracy si nos da información util sobre que tan bueno es un modelo, funciona bien cuando hay una distribución parecida entre las posibles clases (por ejemplo, 50/50 de proporción entre 2 clases "Si" y "No"), así que la estrategia del consultor si puede servir en este caso.

Sin embargo, en el caso de que se desee predecir una clase que esté muy poco presente en comparación a otra clase, el Accuracy puede resultar incertero.

Por ejemplo si 10% de los datos son del tipo "Con Covid" y 90% "Sin Covid", un clasificador que siempre prediga "No" tendrá un Accuracy de 90%, lo cual suena bueno pero no estaríamos prediciendo ningún "Covid" cuando esta etiqueta podría ser importante

Para lidiar con esto, se puedo hacer Macro-avering del coeficiente $F_1$ de cada clase posible, esto le da más representabilidad a las clases que están menos presentes

Estas lineas de código importan las librerias necesarias, cargan la base de datos, hace un DataFrame de Pandas con los atributos y plotea la matriz de correlación entre los atributos mostrando las correlaciones y un color del mapa de calor asociado.

Estre gráfico permite identificar facilmente las variables con mayor dependencia entre sí (valores cercanos a 1 o -1, por ejemplo flavanoids con total_phenols).

Justamente las principales observaciones que se pueden obtener son del tipo "x atributo depende mucho de otro atributo y", por ejemplo, flavanoids con total_phenols tienen una correlación de aproximadamente 0.86, lo que significa que en general una fila con flavanoids alto significa que total_phenols probablemente será alto también.

Más generalmente: si dos atributos (x e y) poseen una correlación cercana a 1, significa que cuando un atributo es alto, el otro tiende a ser alto también.

Si la corr. es cercana a -1, cuando x es elevado y tiende a ser más bajo (y viceversa).

Y si la corr. es cerca de 0, esto nos dice que no hay dependencia entre los atributos.

CLASIFICADOR 3¶

Para este clasificador utilizaremos Cross-validation. Con cross-validation, el conjunto de entrenamiento se divide en $k$ conjuntos disjuntos; se entrena sobre los datos correspondientes a $k-1$ de éstos, y se evalúa sobre el conjunto restante. Esto se repite $k$ veces, evaluando siempre sobre un conjunto distinto. Teniendo en cuenta el parámetro $k$, a éste método se le llama $k$-fold cross-validation.

El código que está a continuación realizará 10-fold cross-validation usando Decision Tree sobre los datos. La forma de estimar el rendimiento del clasificador es, entre otras cosas, calculando el promedio de todos los k-folds.

Además se muestra la predición sobre los datos para testear.

## EJECUTAR ESTE BLOQUE

from sklearn.model_selection import cross_validate
import numpy as np

scoring = ['precision_macro', 'recall_macro', 'accuracy', 'f1_macro']
cv_results = cross_validate(clf, X, y, cv = 10, scoring = scoring, return_train_score= True)

print('Promedio Precision:', np.mean(cv_results['test_precision_macro']))
print('Promedio Recall:', np.mean(cv_results['test_recall_macro']))
print('Promedio F1-score:', np.mean(cv_results['test_f1_macro']))
print('Promedio Accucary:', np.mean(cv_results['test_accuracy']))
